I am interested in adaptive intervention in the mobile health space. I build computational tools that analyze human behavior from phone data (e.g., sensors, self-input) and then intervenes in real time in ways that are personalized and contextualized to users' needs, daily routine, and the environments they live in.

Keywords: Adaptive health intervention, Mobile Sensing, Context-aware computing, Reinforcement learning, User-centered design

Project highlights




MyBehavior

MyBehavior mines a user's dietary intake and physical activity behavior and suggests small personalized changes to these behaviors. MyBehavior is the first app that combined sensor data with reinforcement learning and provided personalized health recommendations.


See our JMIR and Ubicomp 2015 papers for more details on MyBehavior. We also recently extended MyBehavior for chronic back pain and published an article at JMIR.






SARA (Substance Abuse Research Assistant)

SARA (Substance Abuse Research Assistant) provides intervention to increase self-reported data collection in mobile health. SARA tackles the problem of low engagement rates in existing mobile health apps. SARA uses a micro-randomized trial to evaluate a variety of engagement strategies.

See our JMIR paper for more details about SARA.





ReVibe: Recalling daily moments with context

ReVibe tries to increase adherence of Ecological Momentary Assessments (EMAs) by lowering burden. Instead of interrupting at the moment with EMAs, ReVibe asks to recall about daily moments in the evening. ReVibe also provides contextual information (e.g., location, movement) from the same moments to improve recall accuracy.

A manuscript on the ReVibe project is under preparation. But, in a 14-day pilot study with 54 participants, we found contextual information increased recall accuracy by 5.6% and the recall completion rates were 25% higher than EMAs.

MyPersonalCoach

MyPersonalCoach is similar to the MyBehavior project, with an important addon - MyBehavior suggestions are pushed in the right time and context to a smartwatch to make user's active.

The suggestions in the right moment promote two kinds of actions (i) repetition of same action to promote habit building (ii) novel or opportunisitic action to make user's active instantly.

SAINT

Recently in mobile and wearable computing, we have seen a rise in continuous sensing across different types of sensors and in device processing. Combination of these array of sensors and corresponding inference have capability to generate detailed non-trivial multidimensional information about the environment and the individual carrying the device. Also, there is a growing number of third-party applications that use these information for context-aware-computing, personal well-being, and just-in-time health care.

With SAINT, we propose a scalable sensing and real-time sensing and inference toolkit for mobile devices. SAINT uses a client-server architecture. The server can support clients with different sensing (e.g., accelerometer and location) and inference (e.g., human speech) streams on-demand. If more than one client ask for the same sensing or inference data then the server performs necessary computation (e.g., recognizing human speech) once and serves all the clients needing such data with inter process communication. Furthermore, internally SAINT server enables easy sharing of sensing and inference data among different recognition modules (e.g., sleep sensing can use movement and speech inference data) over a simple middle-man >bus like structure. This bus architecture makes any sensing or inference streams available for consumption for any other inference module. This sharing enables complex inference possible by reusing components that has been built before. We show that bus and client server architecture incur low overhead while enabling exponential increase in cpu time and power usage on the number of clients. Also, developers found SAINT framework easy to program and deploy.

MoodRhythm

Bipolar disorder is a common form of mental illness. Medication is the current gold standard for bipolar depression treatment, but medication can have side effect. Social Rhythm Therapy (SRM) is a purely behavioral approach to reduce bipolar depression symptoms. SRM focuses on stabilizing routines to improve biploar symptoms. MoodRhythm is a mHealth version of SRM which facilitates stabilizing routine by monitoring daily routine behaviors and rewarding positive changes.

My role in this project was to infer daily behavior from sensor data.

StressSense

StressSense is a continuous sensing application that captures audio from phone microphones and classifies the audio into stressful or regular conversations. The basic intuition is that, during stressful episodes, the airflow through the vocal cord changes (e.g., pitch, speaking rate, teager energy) and this airflow dynamics can be modeled and classified.


We have an implementation of StressSense on the Android platform, so please let us know if you are interested to try StressSense (mrabbi@fas.harvard.edu).

Mental health sensing

This 2011 project captured fine grained motion and privacy-sensitive audio data from Intel MSP, and derived different metrics that correlated with commonly used surveys for assessing mental well-being.


This project later inspired the BeWell and the StudentLife project.

BeWell

BeWell is a continuous sensing application that infers physical activity, sleep and social interaction from mobile sensor data. BeWell then aggregates the number of minutes of physical activity, social interaction, and sleep in a day, and populates a virtual aquarium in the background of the phone.


BeWell was a milestone project that tied multi-modal sensing with health feedback. It kickstarted a flurry of follow-up projects that include MyBehavior and the StudentLife project at Dartmouth.
